# WebRTC 快速开始指南

## 系统要求

- Python 3.7 或更高版本
- 现代浏览器 (Chrome 80+, Firefox 75+, Edge 80+)
- 摄像头和麦克风 (用于视频通话)

## 安装步骤

### 1. 安装 Python 依赖

```bash
pip install -r requirements.txt
```

或者手动安装:

```bash
pip install aiohttp>=3.8.0
```

### 2. 启动服务器

#### 方法一：使用启动脚本 (Windows)
双击 `start_server.bat`

#### 方法二：命令行启动
```bash
python webrtc_server_for_http.py
```

### 3. 访问应用

服务器启动后，在浏览器中访问:

- **视频通话页面**: http://localhost:8765/webrtc_client.html
- **状态监控页面**: http://localhost:8765/status.html
- **服务器状态 API**: http://localhost:8765/status
- **房间信息 API**: http://localhost:8765/api/rooms

## 使用指南

### 第一次使用

1. **打开视频通话页面**
   - 访问 http://localhost:8765/webrtc_client.html

2. **连接服务器**
   - 点击页面底部的"连接信令服务器"按钮
   - 等待连接成功提示

3. **配置设置 (可选)**
   - 点击"设置"按钮
   - 修改房间号、用户 ID 等
   - 点击"保存"

4. **开始通话**
   - 查看右侧"在线用户"列表
   - 点击用户的"呼叫"按钮
   - 或点击"开始通话"按钮

### 发起通话

**方法一：通过用户列表**
1. 在右侧"在线用户"列表中找到要呼叫的用户
2. 点击该用户旁边的"呼叫"按钮
3. 等待对方接听

**方法二：使用开始通话按钮**
1. 点击"开始通话"按钮
2. 系统会自动呼叫房间中的第一个用户

### 接听通话

1. 当收到呼叫请求时，会弹出通知
2. "接听通话"按钮会变为可用状态
3. 点击"接听 (呼叫方 ID)"按钮
4. 系统会自动建立媒体连接

### 通话控制

- **关闭/开启视频**: 点击本地视频窗口上的"关视频"按钮
- **关闭/开启音频**: 点击本地视频窗口上的"关音频"按钮
- **结束通话**: 点击底部的"挂断通话"按钮

### 查看统计信息

通话过程中，右侧面板会显示实时统计:
- 视频发送/接收带宽
- 视频发送/接收包数量
- 视频帧率
- 音频发送/接收带宽
- RTT (往返时间)

## 常见问题

### Q1: 无法连接服务器
**解决方案**:
1. 确认服务器已启动
2. 检查端口 8765 是否被占用
3. 检查防火墙设置

### Q2: 浏览器提示没有权限访问摄像头/麦克风
**解决方案**:
1. 点击浏览器地址栏的锁形图标
2. 允许访问摄像头和麦克风
3. 刷新页面

### Q3: 看不到自己的视频
**解决方案**:
1. 检查摄像头是否被其他程序占用
2. 在设置中确认视频源选择正确
3. 刷新页面重新获取媒体流

### Q4: 无法呼叫其他用户
**解决方案**:
1. 确认双方都已连接到服务器
2. 确认在同一个房间
3. 检查网络连接

### Q5: 视频卡顿或质量差
**解决方案**:
1. 检查网络带宽
2. 降低视频分辨率 (修改代码中的视频配置)
3. 关闭其他占用带宽的程序

## 配置说明

### 修改服务器端口

编辑 `webrtc_server_for_http.py` 文件，修改配置:

```python
# 在 model/apphelp.py 的 get_config() 中修改
config = {
    "port": 8765  # 修改这里的端口号
}
```

### 修改 STUN 服务器

编辑 `webrtc_client.html` 文件，找到:

```javascript
iceServers: [
    { urls: 'stun:stun.l.google.com:19302' },
    { urls: 'stun:stun1.l.google.com:19302' }
]
```

修改为你自己的 STUN 服务器地址。

### 添加 TURN 服务器 (用于复杂网络环境)

编辑 `webrtc_client.html` 文件，修改 iceServers 配置:

```javascript
iceServers: [
    { urls: 'stun:stun.l.google.com:19302' },
    { 
        urls: 'turn:your-turn-server.com',
        username: 'your-username',
        credential: 'your-password'
    }
]
```

## 开发调试

### 查看服务器日志

服务器日志保存在 `webrtc_server.log` 文件中，可以使用以下命令查看:

```bash
# Windows
type webrtc_server.log

# Linux/Mac
tail -f webrtc_server.log
```

### 浏览器控制台

按 F12 打开浏览器开发者工具，在 Console 标签查看客户端日志。

### 网络监控

在浏览器开发者工具的 Network 标签中:
- 查看 WebSocket 连接
- 查看 WS 消息内容
- 分析网络性能

## 测试多用户

### 方法一：多浏览器窗口
1. 打开多个浏览器窗口
2. 每个窗口访问 http://localhost:8765/webrtc_client.html
3. 在不同窗口中使用不同的用户 ID
4. 加入同一个房间

### 方法二：多浏览器
1. 使用不同的浏览器 (Chrome, Firefox, Edge 等)
2. 每个浏览器访问视频通话页面
3. 配置不同的用户 ID

### 方法三：隐身模式
1. 打开普通窗口
2. 打开隐身/隐私模式窗口
3. 两个窗口使用不同的用户 ID

## 性能优化建议

1. **限制视频分辨率**
   - 在设置中选择较低的视频质量
   - 修改代码中的视频约束

2. **使用有线网络**
   - WiFi 可能不稳定
   - 推荐使用以太网连接

3. **关闭不必要的程序**
   - 减少 CPU 和内存占用
   - 释放网络带宽
